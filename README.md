# 飞书多维表格质检机器人 — 完整部署方案

## 一、整体架构

```
┌─────────────────────────────────────────────────┐
│                   飞书                           │
│                                                  │
│  用户在多维表格中 @质检机器人 + 发送表格链接       │
│              ↓                                    │
│  飞书服务器推送事件到 Webhook URL                  │
└──────────────┬──────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────┐
│          你的电脑（本地运行）                       │
│                                                   │
│  Flask 服务接收请求                                │
│       ↓                                           │
│  解析消息，获取表格链接                            │
│       ↓                                           │
│  调用飞书 API 读取表格数据                         │
│       ↓                                           │
│  质检引擎逐条检查（7 条规则）                      │
│       ↓                                           │
│  生成质检报告                                      │
│       ↓                                           │
│  通过飞书消息发送报告给用户                        │
└──────────────────────────────────────────────────┘
```

## 二、飞书开放平台设置（一步一步操作）

### 第 1 步：创建应用

1. 打开浏览器，访问 https://open.feishu.cn/app
2. 登录你的飞书账号
3. 点击 **「创建自建应用」**
4. 填写：
   - 应用名称：`数据质检机器人`（或你喜欢的名字）
   - 应用描述：`自动检查二期数据质量`
   - 应用图标：随便选一个
5. 点击 **「创建」**
6. 创建后进入应用详情页

### 第 2 步：记录凭证

在应用详情页 → **「凭证与基础信息」** 页面，记录以下信息：

```
App ID:     cli_xxxxxxxxxxxxxx   ← 复制保存
App Secret: xxxxxxxxxxxxxxxxxx   ← 点击显示后复制保存
```

> ⚠️ App Secret 很重要，不要泄露给他人！

### 第 3 步：添加「机器人」能力

1. 在应用详情页，左侧菜单点击 **「应用能力」→「添加应用能力」**
2. 选择 **「机器人」**，点击添加
3. 机器人名称：`质检助手`

### 第 4 步：配置权限

1. 左侧菜单点击 **「权限管理」**
2. 搜索并开通以下权限：

| 权限名称 | 权限标识 | 用途 |
|----------|----------|------|
| 获取与发送单聊消息 | im:message | 发送质检报告 |
| 读取用户发给机器人的消息 | im:message.receive | 接收 @ 消息 |
| 获取用户在机器人所属租户内的信息 | contact:user.id | 识别用户 |
| 查看、评论和导出多维表格 | bitable:app:readonly | 读取表格数据 |
| 查看多维表格 | bitable:app | 读取表格数据 |

> 如果搜不到某个权限，尝试搜索关键词如 "消息"、"多维表格"、"bitable"

### 第 5 步：配置事件订阅

1. 左侧菜单点击 **「事件与回调」→「事件配置」**
2. **请求地址配置**：先跳过，等部署好服务后再填
3. 添加事件：
   - 搜索 `im.message.receive_v1`（接收消息）
   - 勾选并添加

### 第 6 步：配置 Webhook（等服务部署后再做）

等你把下面的代码跑起来后，回来这里填写：
- **请求地址**：`http://你的公网地址:5000/webhook/event`

如果你在本地运行，需要使用 **ngrok** 做内网穿透（详见下方）。

### 第 7 步：发布应用

1. 左侧菜单点击 **「版本管理与发布」**
2. 点击 **「创建版本」**
3. 填写版本号（如 1.0.0）和更新说明
4. 点击 **「保存」→「申请发布」**
5. 如果你是管理员，可以直接审核通过
6. 发布后，机器人就可以被使用了

### 第 8 步：在群组/表格中使用机器人

1. 打开你的飞书多维表格
2. 在任意单元格或评论中 **@质检机器人** 并发送表格链接
3. 机器人会自动读取表格并返回质检报告

## 三、本地部署步骤

### 3.1 安装依赖

```bash
cd C:\Users\edawr\feishu-qa-bot
pip install -r requirements.txt
```

### 3.2 配置环境变量

复制 `.env.example` 为 `.env`，填入你的飞书应用凭证：

```
FEISHU_APP_ID=cli_xxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxx
```

### 3.3 启动服务

```bash
python server.py
```

服务启动后会监听 `http://localhost:5000`

### 3.4 内网穿透（让飞书能访问你的本地服务）

#### 方案 A：ngrok（推荐，免费）

1. 访问 https://ngrok.com 注册账号
2. 下载 ngrok：https://ngrok.com/download
3. 解压后运行：

```bash
ngrok http 5000
```

4. 会显示一个公网地址，类似：
   ```
   Forwarding  https://xxxx-xxx-xxx.ngrok.io → http://localhost:5000
   ```

5. 复制这个 `https://xxxx.ngrok.io` 地址

#### 方案 B：cpolar（国内替代）

1. 访问 https://www.cpolar.com 注册
2. 下载安装后运行：

```bash
cpolar http 5000
```

### 3.5 回到飞书开放平台配置 Webhook

1. 回到飞书开放平台 → 你的应用 → 「事件与回调」
2. 请求地址填入：`https://你的公网地址/webhook/event`
3. 点击「验证」，如果显示成功，说明配置正确

## 六、常见问题

### Q1: 机器人没有回复？
- 检查服务是否在运行
- 检查 ngrok 是否在线
- 检查飞书开放平台事件订阅是否配置正确
- 查看服务日志排查错误

### Q2: 读取表格失败？
- 确认应用权限中已开通「多维表格」相关权限
- 确认表格链接格式正确
- 确认应用已发布

### Q3: ngrok 地址变了？
- ngrok 免费版每次重启地址会变
- 每次重启后需要去飞书开放平台更新 Webhook 地址
- 考虑使用付费版固定地址，或部署到云服务器

### Q4: 想修改质检规则？
- 编辑 `qa_checker.py` 中的规则
- 重启服务即可生效
